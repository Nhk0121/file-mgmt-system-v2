## 系統概述

### 核心架構
文件管理系統 v4 採用**四層結構**：
- **第 1 層**：儲存區（時效區、永久區、資源回收桶）
- **第 2 層**：組別（部門級別）
- **第 3 層**：課別（課室級別）
- **第 4 層**：資料夾與檔案

### 主要功能
- ✅ 時效區：30 天自動清理
- ✅ 永久區：需要審核通過才能看到
- ✅ 權限控制：員工/負責人/超管/外包四個角色
- ✅ 檔案分享：生成分享連結，支援密碼與下載次數限制
- ✅ 組織結構管理：超管可新增/編輯組別與課別
- ✅ 審計日誌：完整的操作紀錄與資安警示

### 軟體需求

| IIS | 10.0 或更新 | Web 伺服器 |

- 內網 SQL Server 連接埠 (預設 1433)
- 建議使用 HTTPS 加密所有通訊

## 資料庫部署

### 步驟 1：建立資料庫

```sql
-- 1. 建立資料庫
CREATE DATABASE [文件管理系統] 
COLLATE Chinese_Taiwan_Stroke_90_CI_AS;

-- 2. 使用新資料庫
USE [文件管理系統];

-- 3. 設定復原模式為完整
ALTER DATABASE [文件管理系統] SET RECOVERY FULL;

-- 4. 設定自動增長
ALTER DATABASE [文件管理系統] 
MODIFY FILE (NAME = N'文件管理系統', FILEGROWTH = 100MB);

ALTER DATABASE [文件管理系統] 
MODIFY FILE (NAME = N'文件管理系統_log', FILEGROWTH = 50MB);
```

### 步驟 2：執行初始化腳本

1. 在 SQL Server Management Studio 中開啟 `資料庫建置.sql`
2. 確保已選擇 `文件管理系統` 資料庫
3. 執行腳本（預期執行時間：5-10 秒）
4. 驗證所有資料表已建立

```sql
-- 驗證資料表
SELECT TABLE_NAME FROM INFORMATION_SCHEMA.TABLES 
WHERE TABLE_SCHEMA = 'dbo' 
ORDER BY TABLE_NAME;
```

### 步驟 3：執行升級腳本

依序執行以下升級腳本：

```sql
-- 1. 執行 v2 升級（帳號欄位、索引）
-- 檔案：資料庫升級_v2.sql

-- 2. 執行 v3 升級（課別表、分享連結表、權限表）
-- 檔案：資料庫升級_v3.sql
```

### 步驟 4：初始化基本資料


## IIS 部署

### 步驟 1：準備應用程式檔案

1. 建立應用程式目錄：`C:\inetpub\wwwroot\文件管理系統`
2. 複製所有 `.aspx` 與 `.cs` 檔案
3. 複製 `App_Code` 目錄（包含所有輔助類別）
4. 複製 `Web.config` 與 `Global.asax`

```
C:\inetpub\wwwroot\文件管理系統\
├── 首頁.aspx
├── 登入.aspx
├── 檔案瀏覽_v2.aspx
├── 檔案上傳_v2.aspx
├── 審核管理.aspx
├── 分享.aspx
├── App_Code\
│   ├── 輔助類別.cs
│   ├── 權限輔助_v2.cs
│   ├── 排程清理輔助.cs
│   ├── ... (其他輔助類別)
├── Web.config
└── Global.asax
```

### 步驟 2：設定 Web.config

編輯 `Web.config` 中的連線字串：

```xml
<configuration>
  <connectionStrings>
    <add name="DefaultConnection" 
         connectionString="Server=localhost;Database=文件管理系統;User Id=sa;Password=YourPassword;" 
         providerName="System.Data.SqlClient" />
  </connectionStrings>
  
  <appSettings>
    <!-- 儲存區根路徑 -->
    <add key="儲存根路徑" value="D:\儲存區" />
    
    <!-- 時效區天數 -->
    <add key="時效區天數" value="30" />
    
    <!-- 最大上傳檔案大小 (MB) -->
    <add key="最大上傳大小" value="100" />
    
    <!-- 允許的檔案副檔名 -->
    <add key="允許副檔名" value=".doc,.docx,.xls,.xlsx,.pdf,.txt,.zip,.rar,.jpg,.png" />
    
    <!-- 審核負責人 IP 設定 -->
    <add key="審核IP白名單" value="192.168.1.100,192.168.1.101" />
  </appSettings>
  
  <system.webServer>
    <!-- 啟用 HTTPS 重導向 -->
    <rewrite>
      <rules>
        <rule name="HTTP to HTTPS" stopProcessing="true">
          <match url="(.*)" />
          <conditions>
            <add input="{HTTPS}" pattern="^OFF$" />
          </conditions>
          <action type="Redirect" url="https://{HTTP_HOST}{REQUEST_URI}" redirectType="Permanent" />
        </rule>
      </rules>
    </rewrite>
  </system.webServer>
</configuration>
```

### 步驟 3：在 IIS 中建立應用程式

1. 開啟 **IIS 管理員**
2. 展開 **Sites** → **Default Web Site**
3. 右鍵點擊 → **Add Application**
4. 設定以下參數：
   - **Alias**: `文件管理系統`
   - **Physical path**: `C:\inetpub\wwwroot\文件管理系統`
   - **Application pool**: 建立新的 `文件管理系統AppPool`

### 步驟 4：設定應用程式池

1. 在 IIS 中選擇 **Application Pools**
2. 右鍵點擊 `文件管理系統AppPool` → **Edit**
3. 設定以下參數：
   - **.NET CLR version**: `.NET CLR Version v4.0.30319`
   - **Managed pipeline mode**: `Integrated`
   - **Identity**: `ApplicationPoolIdentity`

### 步驟 5：設定檔案夾權限

```powershell
# 以管理員身份執行 PowerShell

# 設定應用程式目錄權限
$AppPath = "C:\inetpub\wwwroot\文件管理系統"
$AppPool = "IIS AppPool\文件管理系統AppPool"

# 新增讀取與寫入權限
$acl = Get-Acl $AppPath
$rule = New-Object System.Security.AccessControl.FileSystemAccessRule(
    $AppPool, "Modify", "ContainerInherit,ObjectInherit", "None", "Allow"
)
$acl.AddAccessRule($rule)
Set-Acl -Path $AppPath -AclObject $acl

# 設定儲存區目錄權限
$StoragePath = "D:\儲存區"
$acl = Get-Acl $StoragePath
$rule = New-Object System.Security.AccessControl.FileSystemAccessRule(
    $AppPool, "Modify", "ContainerInherit,ObjectInherit", "None", "Allow"
)
$acl.AddAccessRule($rule)
Set-Acl -Path $StoragePath -AclObject $acl
```

### 步驟 6：建立儲存區目錄

```powershell
# 建立儲存區目錄結構
$StoragePath = "D:\儲存區"

# 時效區
New-Item -Path "$StoragePath\時效區" -ItemType Directory -Force
New-Item -Path "$StoragePath\時效區\G001\D001" -ItemType Directory -Force

# 永久區
New-Item -Path "$StoragePath\永久區" -ItemType Directory -Force
New-Item -Path "$StoragePath\永久區\G001\D001" -ItemType Directory -Force

# 資源回收桶
New-Item -Path "$StoragePath\資源回收桶" -ItemType Directory -Force
```

### 步驟 7：配置 HTTPS

1. 在 IIS 中選擇應用程式
2. 雙擊 **SSL Settings**
3. 勾選 **Require SSL**
4. 選擇有效的 SSL 憑證

## 系統配置

### 步驟 1：初始化排程清理

編輯 `Global.asax`：

```csharp
protected void Application_Start()
{
    // 初始化排程清理任務
    排程清理配置.初始化排程();
    
    // 其他初始化代碼...
}
```

### 步驟 2：設定 SQL Server 代理排程（可選）

在 SQL Server Management Studio 中建立 SQL Agent Job：

```sql
-- 建立排程清理 Job
USE msdb;
GO

EXEC sp_add_job @job_name = N'文件管理系統_時效區清理';

EXEC sp_add_jobstep 
    @job_name = N'文件管理系統_時效區清理',
    @step_name = N'執行清理',
    @subsystem = N'TSQL',
    @command = N'
        -- 時效區清理 SQL
        UPDATE 檔案主檔 
        SET 儲存區類型=N''資源回收桶'', 是否刪除=1, 刪除時間=GETDATE()
        WHERE 儲存區類型=N''時效區'' 
        AND 是否刪除=0
        AND 到期時間 < GETDATE();
    ';

-- 每天凌晨 2 點執行
EXEC sp_add_schedule 
    @schedule_name = N'每日凌晨2點',
    @freq_type = 4,
    @freq_interval = 1,
    @active_start_time = 020000;

EXEC sp_attach_schedule 
    @job_name = N'文件管理系統_時效區清理',
    @schedule_name = N'每日凌晨2點';
```

### 步驟 3：設定備份策略

```sql
-- 建立完整備份 Job
BACKUP DATABASE [文件管理系統] 
TO DISK = N'D:\備份\文件管理系統_完整_$(DATE).bak'
WITH INIT, COMPRESSION;

-- 建立交易日誌備份 Job (每小時執行)
BACKUP LOG [文件管理系統]

WITH INIT, COMPRESSION;
```

## 故障排除

### 常見問題

#### Q1：無法連線到資料庫
**症狀**：登入頁面顯示「資料庫連線失敗」

**解決方案**：
1. 檢查 SQL Server 是否啟動：`Services.msc` → SQL Server
2. 檢查連線字串：`Web.config` 中的 Server 與 Database 名稱
3. 檢查 SQL Server 防火牆：開放 1433 連接埠
4. 檢查帳號權限：確保 SQL Server 帳號有資料庫存取權限

```sql
-- 測試連線
sqlcmd -S localhost -U sa -P YourPassword -d 文件管理系統
```

#### Q2：檔案上傳失敗
**症狀**：上傳檔案時顯示「上傳失敗」

**解決方案**：
1. 檢查儲存區目錄權限：`icacls D:\儲存區`
2. 檢查磁碟空間：`dir D:\` 查看可用空間
3. 檢查檔案大小限制：`Web.config` 中的 `最大上傳大小`
4. 檢查副檔名限制：`Web.config` 中的 `允許副檔名`

#### Q3：HTTPS 連線失敗
**症狀**：瀏覽器顯示「不安全的連線」

**解決方案**：
1. 檢查 SSL 憑證有效期：`certmgr.msc`
2. 檢查 IIS 中的 SSL 設定
3. 重新啟動 IIS：`iisreset`

#### Q4：時效區清理未執行
**症狀**：過期檔案未自動移到回收桶

**解決方案**：
1. 檢查 `Global.asax` 是否呼叫了 `排程清理配置.初始化排程()`
2. 檢查 IIS 應用程式池是否正常運行
3. 手動執行清理：在系統管理員頁面點擊「手動清理」按鈕

```csharp
// 手動觸發清理
排程清理輔助.執行時效區清理();
排程清理輔助.執行回收桶清理(30);
排程清理輔助.清理過期分享連結();
```

### 監控指標

```sql
-- 監控資料庫大小
SELECT 
    DB_NAME() AS 資料庫名稱,
    CAST(SUM(size) * 8.0 / 1024 / 1024 AS DECIMAL(10,2)) AS 大小_GB
FROM sys.master_files
WHERE database_id = DB_ID()
GROUP BY database_id;

-- 監控檔案數量
SELECT 
    儲存區類型,
    COUNT(*) AS 檔案數量,
    CAST(SUM(檔案大小) / 1024 / 1024 / 1024.0 AS DECIMAL(10,2)) AS 總大小_GB
FROM 檔案主檔
WHERE 是否刪除 = 0
GROUP BY 儲存區類型;

-- 監控清理紀錄
SELECT TOP 10
    清理時間,
    清理類型,
    清理狀態,
    備註
FROM 排程清理紀錄
ORDER BY 清理時間 DESC;
```
