## 前置準備

### 需要準備的物品

| 項目 | 說明 | 備註 |
|------|------|------|
| **安裝檔案** | 文件管理系統源碼 | 應包含 App_Code、aspx 檔案等 |
| **資料庫腳本** | `資料庫建置.sql` + `資料庫升級_v2.sql` | 用於建立與升級資料庫 |
| **配置文件** | `Web.config` 範本 | 需根據環境修改 |
| **磁碟空間** | 至少 20GB | 用於儲存檔案 |
| **網路連接** | 穩定的網路 | 用於資料庫連接 |

### 所需軟體版本

| 軟體 | 最低版本 | 推薦版本 | 檢查方式 |
|------|---------|---------|---------|
| Windows Server | 2012 R2 | 2019 或 2022 | 右鍵「此電腦」→ 內容 |
| SQL Server | 2012 | 2016 或 2019 | SQL Server Management Studio |
| .NET Framework | 4.5 | 4.8 | 控制台 → 程式 → 程式和功能 |
| IIS | 7.5 | 10.0 | 伺服器管理員 → 角色與功能 |

### 帳號與權限

| 帳號 | 用途 | 權限 |
|------|------|------|
| **Windows 管理員** | 安裝軟體、配置 IIS | 本機系統管理員 |
| **SQL Server SA** | 建立資料庫、執行腳本 | SQL Server 系統管理員 |
| **應用程式池帳號** | 運行 ASP.NET 應用程式 | 讀取應用程式檔案、寫入日誌 |

## 環境檢查

### 步驟 1：檢查 Windows Server 版本

**操作步驟**：
1. 按 `Win + R` 打開「執行」對話框
2. 輸入 `winver` 並按 Enter
3. 查看 Windows 版本號

**預期結果**：Windows Server 2012 R2 或更新版本

**如果版本過舊**：
- 聯絡 IT 部門進行系統升級
- 或在新的伺服器上部署

---

### 步驟 2：檢查 .NET Framework 版本

**操作步驟**：
1. 按 `Win + R` 打開「執行」對話框
2. 輸入 `regedit` 打開登錄編輯程式
3. 導航至：`HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full`
4. 查看 `Release` 值

**版本對照表**：
| Release 值 | .NET Framework 版本 |
|-----------|-------------------|
| 378389 | 4.5 |
| 378758 | 4.5.1 |
| 379893 | 4.5.2 |
| 393295 | 4.6 |
| 394254 | 4.6.1 |
| 394802 | 4.6.2 |
| 460798 | 4.7 |
| 461308 | 4.7.1 |
| 461814 | 4.7.2 |
| 528040 | 4.8 |

**如果版本低於 4.5**：
- 下載並安裝 .NET Framework 4.8（推薦）
- 下載地址：https://dotnet.microsoft.com/download/dotnet-framework

---

### 步驟 3：檢查 SQL Server 版本

**操作步驟**：
1. 打開 SQL Server Management Studio
2. 連接到您的 SQL Server 實例
3. 在「物件總管」中右鍵點擊伺服器名稱
4. 選擇「內容」查看版本

**預期結果**：SQL Server 2012 或更新版本

**如果未安裝 SQL Server**：
- 下載並安裝 SQL Server Express（免費）或 SQL Server Standard
- 下載地址：https://www.microsoft.com/sql-server/sql-server-downloads

---

### 步驟 4：檢查 IIS 是否已安裝

**操作步驟**：
1. 按 `Win + R` 打開「執行」對話框
2. 輸入 `inetmgr` 打開 IIS 管理員
3. 如果成功打開，表示 IIS 已安裝

**如果 IIS 未安裝**：
1. 打開「伺服器管理員」
2. 點擊「新增角色和功能」
3. 選擇「Web 伺服器 (IIS)」
4. 完成安裝向導

## 資料庫部署

### 步驟 1：建立資料庫

**操作步驟**：

1. **打開 SQL Server Management Studio**
   - 按 `Win + R`，輸入 `ssms`，按 Enter
   - 或從「開始」菜單找到 SQL Server Management Studio

2. **連接到 SQL Server**
   - 伺服器名稱：輸入您的 SQL Server 實例名稱（例如 `DESKTOP-ABC\SQLEXPRESS`）
   - 驗證：選擇「Windows 驗證」或「SQL Server 驗證」
   - 點擊「連接」

3. **建立資料庫**
   - 在「物件總管」中右鍵點擊「資料庫」
   - 選擇「新增資料庫」
   - 資料庫名稱：輸入 `文件管理系統`
   - 點擊「確定」

4. **執行建置腳本**
   - 打開「新增查詢」窗口
   - 複製 `資料庫建置.sql` 的全部內容
   - 粘貼到查詢窗口
   - 點擊「執行」（或按 F5）

5. **執行升級腳本**
   - 打開「新增查詢」窗口
   - 複製 `資料庫升級_v2.sql` 的全部內容
   - 粘貼到查詢窗口
   - 點擊「執行」

**驗證結果**：
- 在「物件總管」中展開「文件管理系統」資料庫
- 應該看到以下資料表：
  - 使用者帳號
  - 組別設定
  - 資料夾
  - 檔案主檔
  - 操作紀錄
  - 資安稽核紀錄
  - 備份設定

---

### 步驟 2：建立資料庫登入帳號

**操作步驟**：

1. **建立 SQL Server 登入**
   - 在「物件總管」中展開「安全性」
   - 右鍵點擊「登入」
   - 選擇「新增登入」

2. **填寫登入資訊**
   - 登入名稱：`FileManagementApp`
   - 驗證：選擇「SQL Server 驗證」
   - 密碼：輸入強密碼（例如 `P@ssw0rd123!`）
   - 確認密碼：重新輸入密碼
   - 點擊「確定」

3. **設定資料庫權限**
   - 展開「安全性」→「登入」
   - 右鍵點擊 `FileManagementApp`
   - 選擇「內容」
   - 進入「使用者對應」頁籤
   - 勾選「文件管理系統」
   - 在下方勾選以下資料庫角色：
     - `db_owner`（資料庫擁有者）
   - 點擊「確定」

**記錄連接字串**：
```
Server=DESKTOP-ABC\SQLEXPRESS;Database=文件管理系統;User Id=FileManagementApp;Password=P@ssw0rd123!;
```
（將 `DESKTOP-ABC\SQLEXPRESS` 替換為您的 SQL Server 實例名稱）

## IIS 配置

### 步驟 1：建立應用程式池

**操作步驟**：

1. **打開 IIS 管理員**
   - 按 `Win + R`，輸入 `inetmgr`，按 Enter

2. **建立應用程式池**
   - 在左側樹狀結構中，展開您的伺服器名稱
   - 右鍵點擊「應用程式池」
   - 選擇「新增應用程式池」

3. **填寫應用程式池設定**
   - 名稱：`FileManagementPool`
   - .NET Framework 版本：`.NET Framework v4.0.30319`
   - 受管管道模式：`整合式`
   - 點擊「確定」

4. **設定應用程式池身分**
   - 右鍵點擊 `FileManagementPool`
   - 選擇「進階設定」
   - 找到「身分」選項
   - 點擊「...」按鈕
   - 選擇「自訂帳號」
   - 點擊「設定」
   - 輸入 Windows 帳號（例如 `DOMAIN\AppPool`）和密碼
   - 點擊「確定」

---

### 步驟 2：建立網站

**操作步驟**：

1. **建立網站**
   - 在 IIS 管理員中，右鍵點擊「網站」
   - 選擇「新增網站」

2. **填寫網站設定**
   - 網站名稱：`FileManagementSystem`
   - 應用程式池：選擇 `FileManagementPool`
   - 實體路徑：選擇應用程式檔案所在的目錄（例如 `C:\inetpub\wwwroot\FileManagement`）
   - 繫結：
     - 類型：`http`
     - IP 位址：`全部未指派`
     - 連接埠：`80`
     - 主機名稱：留空
   - 點擊「確定」

3. **驗證網站**
   - 在 IIS 管理員中選擇您的網站
   - 在右側操作面板中點擊「瀏覽 *:80`
   - 應該看到應用程式的首頁

---

### 步驟 3：配置 HTTPS（推薦）

**操作步驟**：

1. **取得 SSL 憑證**
   - 向受信任的憑證頒發機構購買 SSL 憑證
   - 或使用自簽憑證（僅用於測試環境）

2. **匯入 SSL 憑證**
   - 在 IIS 管理員中，選擇伺服器名稱
   - 雙擊「伺服器憑證」
   - 在右側操作面板中點擊「匯入」
   - 選擇憑證檔案（.pfx 或 .cer）
   - 輸入密碼（如果有）
   - 點擊「確定」

3. **為網站新增 HTTPS 繫結**
   - 選擇您的網站
   - 在右側操作面板中點擊「繫結」
   - 點擊「新增」
   - 類型：選擇 `https`
   - IP 位址：`全部未指派`
   - 連接埠：`443`
   - SSL 憑證：選擇您匯入的憑證
   - 點擊「確定」

---

## 應用程式部署

### 步驟 1：準備應用程式檔案

**操作步驟**：

1. **建立應用程式目錄**
   - 打開「檔案總管」
   - 導航至 `C:\inetpub\wwwroot\`
   - 建立新資料夾 `FileManagement`

2. **複製應用程式檔案**
   - 從來源位置複製所有 ASP.NET 檔案
   - 包括：
     - `*.aspx` 檔案（頁面檔案）
     - `*.aspx.cs` 檔案（程式碼後置檔案）
     - `App_Code` 資料夾（輔助類別）
     - `bin` 資料夾（編譯的組件）
     - `Web.config` 檔案（配置檔案）
   - 粘貼到 `C:\inetpub\wwwroot\FileManagement`

---

### 步驟 2：修改 Web.config 配置

**操作步驟**：

1. **打開 Web.config**
   - 用記事本或 Visual Studio Code 打開 `Web.config`

2. **修改連接字串**
   ```xml
   <connectionStrings>
     <add name="DefaultConnection" 
          connectionString="Server=DESKTOP-ABC\SQLEXPRESS;Database=文件管理系統;User Id=FileManagementApp;Password=P@ssw0rd123!;" 
          providerName="System.Data.SqlClient" />
   </connectionStrings>
   ```
   - 將 `DESKTOP-ABC\SQLEXPRESS` 替換為您的 SQL Server 實例名稱
   - 將 `FileManagementApp` 替換為您建立的資料庫登入帳號
   - 將 `P@ssw0rd123!` 替換為對應的密碼

3. **修改應用程式設定**
   ```xml
   <appSettings>
     <!-- 儲存路徑設定 -->
     <add key="永久區路徑" value="D:\FileStorage\永久區\" />
     <add key="時效區路徑" value="D:\FileStorage\時效區\" />
     <add key="資源回收桶路徑" value="D:\FileStorage\回收桶\" />
     <add key="儲存根路徑" value="D:\FileStorage\" />
     
     <!-- 系統參數 -->
     <add key="時效區天數" value="30" />
     <add key="最大上傳MB" value="500" />
     <add key="系統名稱" value="企業文件管理系統" />
     <add key="回收桶清除天數" value="60" />
   </appSettings>
   ```
   - 根據您的實際磁碟路徑修改儲存路徑
   - 根據業務需求修改系統參數

4. **保存檔案**
   - 按 `Ctrl + S` 保存

---

### 步驟 3：建立儲存目錄

**操作步驟**：

1. **建立主儲存目錄**
   - 打開「檔案總管」
   - 導航至 `D:\`（或其他有足夠空間的磁碟）
   - 建立資料夾 `FileStorage`

2. **建立子目錄**
   - 在 `FileStorage` 中建立以下資料夾：
     - `永久區`
     - `時效區`
     - `回收桶`

3. **設定資料夾權限**
   - 右鍵點擊 `FileStorage` 資料夾
   - 選擇「內容」→「安全性」
   - 點擊「編輯」
   - 選擇應用程式池帳號（例如 `DOMAIN\AppPool`）
   - 勾選「完全控制」
   - 點擊「套用」→「確定」

---

### 步驟 4：編譯應用程式（如果需要）

**操作步驟**：

1. **如果有 Visual Studio**
   - 打開 Visual Studio
   - 打開專案檔案（`.csproj` 或 `.vbproj`）
   - 選擇「建置」→「建置方案」
   - 等待編譯完成

2. **如果沒有 Visual Studio**
   - 使用命令列編譯器（已包含在 .NET Framework 中）
   - 打開「命令提示字元」（以管理員身分）
   - 導航至應用程式目錄
   - 執行編譯命令（需要根據您的專案類型調整）

## 系統測試

### 步驟 1：驗證資料庫連接

**操作步驟**：

1. **打開應用程式首頁**
   - 在瀏覽器中輸入 `http://localhost/FileManagement/`
   - 或 `https://localhost/FileManagement/`（如果配置了 HTTPS）

2. **檢查是否出現錯誤**
   - 如果看到應用程式首頁，表示連接成功
   - 如果看到錯誤訊息，請查看[故障排除](#故障排除)部分

---

### 步驟 2：測試登入功能

**操作步驟**：

1. **使用預設帳號登入**
   - 帳號：`admin`
   - 密碼：`admin123`
   - 點擊「登入」

2. **驗證登入結果**
   - 成功登入後應該看到首頁統計面板
   - 如果登入失敗，檢查帳號是否已在資料庫中建立

---

### 步驟 3：測試檔案上傳

**操作步驟**：

1. **導航到「檔案上傳」頁面**
   - 點擊左側菜單中的「檔案上傳」

2. **上傳測試檔案**
   - 選擇一個小檔案（例如 1MB 以下）
   - 選擇目標資料夾
   - 點擊「上傳」

3. **驗證上傳結果**
   - 上傳成功後應該看到確認訊息
   - 檢查檔案是否出現在「檔案瀏覽」頁面

---

### 步驟 4：測試其他核心功能

| 功能 | 測試步驟 | 預期結果 |
|------|---------|--------|
| **檔案瀏覽** | 點擊「檔案瀏覽」，查看資料夾與檔案列表 | 能看到已上傳的檔案 |

## 故障排除

### 問題 1：無法連接資料庫

**症狀**：
- 打開應用程式時出現「無法連接到資料庫」錯誤
- 或看到 SQL Server 連接錯誤訊息

**排除步驟**：

1. **驗證 SQL Server 是否運行**
   - 打開「服務」（`services.msc`）
   - 查找 `SQL Server (SQLEXPRESS)` 或您的實例名稱
   - 確保狀態為「執行中」

2. **驗證連接字串**
   - 打開 `Web.config`
   - 檢查連接字串中的伺服器名稱、帳號、密碼是否正確
   - 使用 SQL Server Management Studio 測試連接

3. **檢查資料庫登入帳號**
   - 在 SQL Server Management Studio 中，驗證 `FileManagementApp` 帳號是否存在
   - 驗證帳號是否有「文件管理系統」資料庫的存取權限

4. **檢查防火牆設定**
   - 確保 SQL Server 連接埠（預設 1433）未被防火牆阻止
   - 或在防火牆中新增例外規則

---

### 問題 2：應用程式無法啟動

**症狀**：
- 打開應用程式時出現「500 Internal Server Error」
- 或看到「應用程式錯誤」訊息

**排除步驟**：

1. **檢查 IIS 日誌**
   - 打開 IIS 管理員
   - 選擇您的網站
   - 雙擊「日誌」
   - 查看最新的錯誤訊息

2. **啟用詳細錯誤訊息**
   - 在 IIS 管理員中，選擇您的網站
   - 雙擊「錯誤頁面」
   - 在右側操作面板中點擊「編輯功能設定」
   - 選擇「詳細錯誤」
   - 點擊「確定」

3. **檢查應用程式檔案**
   - 確保所有 `.aspx` 和 `.cs` 檔案都已複製到應用程式目錄
   - 確保 `Web.config` 檔案存在且格式正確

4. **檢查 .NET Framework 版本**
   - 確保應用程式池的 .NET Framework 版本與應用程式相匹配
   - 如果應用程式是 .NET 4.0，應用程式池應設定為 `.NET Framework v4.0.30319`

---

### 問題 3：無法上傳檔案

**症狀**：
- 上傳檔案時出現錯誤
- 或檔案上傳後無法在檔案瀏覽中看到

**排除步驟**：

1. **檢查儲存目錄權限**
   - 確保應用程式池帳號有「儲存目錄」的讀寫權限
   - 右鍵點擊儲存目錄 → 內容 → 安全性
   - 確保應用程式池帳號有「完全控制」權限

2. **檢查磁碟空間**
   - 確保儲存磁碟有足夠的空間
   - 打開「磁碟管理」檢查可用空間

3. **檢查檔案大小限制**
   - 在 `Web.config` 中查看 `最大上傳MB` 設定
   - 確保上傳的檔案大小不超過限制

4. **檢查上傳路徑**
   - 在 `Web.config` 中驗證上傳路徑設定是否正確
   - 確保路徑存在且可寫

---

### 問題 4：登入失敗

**症狀**：
- 輸入帳號密碼後無法登入
- 或看到「帳號或密碼錯誤」訊息

**排除步驟**：

1. **驗證帳號是否存在**
   - 打開 SQL Server Management Studio
   - 執行查詢：`SELECT * FROM 使用者帳號 WHERE 登入帳號='admin'`
   - 確保帳號存在

2. **驗證帳號狀態**
   - 查看帳號的 `是否啟用` 和 `是否鎖定` 欄位
   - 確保帳號已啟用且未被鎖定

3. **重設密碼**
   - 如果忘記密碼，可以在資料庫中直接更新
   - 執行查詢：`UPDATE 使用者帳號 SET 密碼='新密碼' WHERE 登入帳號='admin'`
   - 注意：新密碼應該是已加密的值（使用 SHA256 + Salt）

4. **檢查 Session 設定**
   - 在 `Web.config` 中驗證 Session 超時設定
   - 預設應為 30 分鐘

### Q1：如何修改系統名稱？

**A**：在 `Web.config` 中修改 `系統名稱` 設定：
```xml
<add key="系統名稱" value="我的企業文件管理系統" />
```

### Q2：如何修改時效區天數？

**A**：在 `Web.config` 中修改 `時效區天數` 設定：
```xml
<add key="時效區天數" value="60" />
```
（表示檔案在時效區保留 60 天）

---

### Q3：如何新增新帳號？

**A**：
1. 以超管帳號登入系統
2. 點擊「帳號管理」
3. 點擊「新增帳號」
4. 填寫帳號資訊並保存

或直接在資料庫中插入：
```sql
INSERT INTO 使用者帳號 (登入帳號, 密碼, 姓名, 角色, 組別編號, 是否啟用)
VALUES ('newuser', 'encrypted_password', '新使用者', '員工', 1, 1)
```

---

### Q4：如何備份資料庫？

**A**：
1. 打開 SQL Server Management Studio
2. 在「物件總管」中右鍵點擊「文件管理系統」資料庫
3. 選擇「工作」→「備份」
4. 選擇備份類型（完整備份）
5. 指定備份檔案位置
6. 點擊「確定」

---

### Q5：如何還原資料庫？

**A**：
1. 打開 SQL Server Management Studio
2. 在「物件總管」中右鍵點擊「資料庫」
3. 選擇「還原資料庫」
4. 選擇備份檔案
5. 點擊「確定」

---

### Q7：如何提高系統安全性？

**A**：
1. **啟用 HTTPS** - 加密傳輸資料
2. **定期修改密碼** - 鼓勵使用者定期修改密碼
3. **限制登入失敗次數** - 防止暴力破解
4. **定期備份資料庫** - 防止資料丟失
5. **監控操作紀錄** - 定期檢查審計日誌
6. **限制檔案上傳類型** - 防止惡意檔案上傳
