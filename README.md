# 文件管理系統（改良版 v2）

企業內部文件管理系統，基於 ASP.NET WebForms + SQL Server，支援多組別、多儲存區、IP 白名單、審核流程與完整稽核紀錄。

---

## 改良摘要（v2）

### A 類：安全性修補

| 項目 | 改良前 | 改良後 |
|------|--------|--------|
| **審核身分驗證** | 依 IP 判斷（可偽造） | 改用 Session 帳號角色，管理員可審全部，負責人只審本組 |
| **上傳安全檢查** | 各頁面自行實作，有遺漏 | 統一呼叫 `檔案安全輔助.驗證上傳檔案()`，含雙重副檔名、MIME 比對 |
| **審核 SQL** | 字串拼接 `IN (...)`，有 SQL Injection 風險 | 改為只含整數 ID 的安全拼接 |
| **角色保護方法** | 缺少 `要求負責人以上()` | 新增統一入口方法 |

### B 類：資料一致性修補

| 項目 | 改良前 | 改良後 |
|------|--------|--------|
| **資料夾改名** | 只改 DB 名稱，實體路徑與子檔案路徑不同步 | 新增 `重新命名資料夾()`：先移磁碟目錄，遞迴同步所有子孫路徑 |
| **審核移檔失敗** | catch 後仍更新 DB 路徑（DB 與磁碟不一致） | 移檔失敗只記錄警示，DB 路徑維持原值 |
| **回收桶還原** | 只改 DB 狀態，實體檔案仍在回收桶（下載會失敗） | 補實體移檔至時效區，同步更新 DB 路徑 |

### C 類：功能補強

| 項目 | 改良前 | 改良後 |
|------|--------|--------|
| **操作紀錄** | 部分路徑只記 IP | 全面記錄 `帳號編號` 與 `登入帳號` |
| **首頁統計** | 所有人看全系統數據 | 非管理員只看自己組別的統計 |
| **最近操作顯示** | 只顯示 IP | JOIN 使用者帳號，顯示操作者姓名 |
| **上傳/還原天數** | 寫死「30天」 | 從 `AppSettings["時效區天數"]` 動態讀取 |

## 目錄結構

```
├── App_Code/
│   ├── 帳號輔助.cs         # 帳號/角色輔助（含 要求負責人以上）
│   ├── 資料夾輔助.cs       # 資料夾管理（含 重新命名資料夾 同步路徑）
│   ├── 安全輔助.cs         # 登入防護 + 上傳安全檢查
│   ├── 輔助類別.cs         # DB/操作紀錄/IP/個資偵測輔助
│   ├── 設定輔助.cs         # 系統設定讀取
│   └── GlobalApplication.cs
├── Handlers/
│   ├── 資料夾內容.ashx     # AJAX 資料夾內容
│   └── 資料夾樹.ashx       # AJAX 資料夾樹
├── 首頁.aspx / .cs
├── 登入.aspx / .cs
├── 忘記密碼.aspx / .cs
├── 檔案瀏覽.aspx / .cs
├── 檔案上傳.aspx / .cs
├── 審核管理.aspx / .cs
├── 資源回收桶.aspx / .cs
├── 錯誤.aspx
├── Web.config
├── 資料庫建置.sql          # 初始建置腳本
└── 資料庫升級_v2.sql       # v2 改良對應的 DB 升級腳本
```

---

## 部署步驟

1. 在 SQL Server 執行 `資料庫建置.sql`（新部署）或 `資料庫升級_v2.sql`（從 v1 升級）
2. 修改 `Web.config` 中的資料庫連線字串與儲存路徑
3. 部署至 IIS（.NET Framework 4.x）
4. 首次登入帳號：`000000`，密碼：`Admin@123456`（系統自動初始化）

> **注意**：執行升級腳本前請先備份資料庫。腳本採用 `IF NOT EXISTS` 保護，可安全重複執行。

---

## 技術規格

- **框架**：ASP.NET WebForms (.NET Framework 4.x)
- **資料庫**：SQL Server（支援 MSSQL 2016+）
- **驗證**：SHA-256 + Salt 雜湊密碼
- **稽核**：操作紀錄、資安稽核、個資偵測三層記錄
- **儲存區**：永久區（需審核）、時效區（自動到期）、資源回收桶
